# AGENTS.md — funx

This file provides context for AI agents and automated tooling that interact with the `funx` project.

## Project overview

`funx` is a Dart package of composable function decorators. It wraps ordinary functions in small, chainable, reusable building blocks that add single concerns — timing, concurrency, reliability, caching, observability, and more — without rewriting the function itself.

- **Language**: Dart
- **License**: MIT
- **Current version**: 1.3.1
- **Author**: Maksymilian Jakcowski

## Repository layout

- `lib/` — Dart source code for the package.
- `doc/` — Markdown documentation source of truth.
- `website/` — Hugo site that publishes the documentation.
- `test/` — Unit and integration tests.
- `tool/` — Developer tooling, including docs validation.

## Common tasks

- Run tests: `dart test`
- Analyze: `dart analyze --fatal-infos`
- Validate docs snippets: `dart test tool/docs_validation_test.dart`
- Build website: `cd website && dart prepare_content.dart && hugo --gc --minify`
- Serve website locally: `cd website && hugo server --bind 127.0.0.1 --port 1313`

## Notes for agents

- Documentation source of truth is the `doc/` folder. The `website/content/docs/` directory is generated by `website/prepare_content.dart` and should not be edited directly.
- The site uses the LotusDocs Hugo theme consumed as a Hugo module.
- Avoid running `git commit`, `git push`, `git reset`, or `git rebase` unless explicitly asked.
